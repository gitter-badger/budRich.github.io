Or, how I became a tiler

i3 is a dynamic window manager for X. I use i3, and I really like it. There are many similar window managers and I haven’t tried many of them. I have run and tinkered with bug.n on windows and tested awesome and dwm on GNU/Linux.

Maybe I was unlucky and had bad timing with awesome, I was very fresh on GNU/Linux and I think the awesome project was in a transition phase, but I had a hard time grasping all of it. I think the biggest downside (after the naming) with awesome is that the project is too much. It has window decoration, bars and widgets and a a lot of different tiling modes. It was at least too much for me. At least back then. I am sure I would look at it different today.

So when I wanted to try an alternative, i3 was the first choice, it looked easy and actively developed and improved. It also had a sane, well written, single page, userguide with dark background.

It turned out that i3 was the opposite of awesome in it’s design philosophy. All i3 does is managing windows, (almost) everything else is handled by external programs. There didn’t seem to be an obvious way to write plugins or scripts for it. All configuration was done in a more or less static one page config.

The i3 devs are very conservative when it comes to functionality. This made me feel that I could learn and understand the program and not get irritated on inconsistent design, breaking changes or broken abandoned plugins.

Two years later, I am still finding new ways to use this fantastic program. i3 have support for tabbed (and stacked),floating and tiled (a dynamic window manager) containers. After some digging I found that you could do much more then the basic commands via i3-msg. It seems to get more popular by every day and it even ships with some distributions now.

The best thing with becoming a tiler is that all the distro-flamewar-BS that poisons the GNU/Linux community in general, becomes totally irrelevant.

The worst thing is that there is no turning back. Almost all computers, other then your own, will have a non-tiling WM. It is actually hard to use them without getting annoyed. And this is a serious downside, worth considering…
